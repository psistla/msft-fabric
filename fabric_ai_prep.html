Four capabilities that changed what's possible

Microsoft Fabric's semantic layer has moved from a reporting tool to an AI execution surface. These are the capabilities that matter for regulated-industry clients right now.

GA · Nov 2025
Prep Data for AI
Three-part semantic enrichment for AI agents
The semantic model now has a dedicated AI configuration surface with three components: an AI Data Schema that scopes which tables and columns the AI can see; Verified Answers that pin specific questions to pre-approved visuals; and AI Instructions that act as a system prompt for the model. All three configurations are automatically honored by Fabric Data Agents and Copilot — configure once, both benefit.
Unlocks: Copilot + Data Agent accuracy Semantic Model
GA · Nov 2025
Fabric Data Agent
Natural language to SQL, DAX, and KQL — governed
Data Agents (formerly AI Skills) convert plain-English questions into native queries across lakehouses, warehouses, eventhouses, and semantic models. Critically for regulated clients: permissions set in the data platform are fully enforced — the agent cannot return data the user cannot see directly. Now supports SQL database artifacts directly, eliminating intermediate lakehouse steps. Instruction character limits expanded to 15,000 for complex domain models.
Unlocks: Self-service without PHI exposure Data Agent
GA · March 2026
Purview + Fabric Integration
Unified security and governance at the semantic layer
New Purview innovations announced March 2026 prevent data oversharing specifically within AI retrieval scenarios. Sensitivity labels now flow from the semantic model into every agent response, Copilot output, and exported artifact. AI-generated answers inherit the classification of the data used to produce them — a critical control for HIPAA and PCI DSS environments where outputs are as regulated as inputs.
Unlocks: Compliant AI output, not just compliant data Governance

Why this matters for regulated clients
The semantic layer is now the AI security perimeter
Strategic shift
Before
AI queries bypassed semantic model controls. Security was applied at the reporting layer only. AI could return data users couldn't see in reports.
Now
RLS, CLS, sensitivity labels, and Purview access policies are all enforced at the agent query layer. Security travels with the data through to the AI response.
Implication
The semantic model is no longer just a reporting optimization. It is the governed, classified, access-controlled interface for all AI workloads.
Verify before the workshop
Verified Answers in Prep for AI is not available in DirectLake mode — only Import and DirectQuery. Confirm the client's intended connection mode before promising this capability. FabCon 2026 GA of Data Agents supersedes any earlier "preview" caveats in your documentation.
Best Practices

How to get data ready for AI workloads

Structured as a sequence. Each row is a distinct practice area with the action required, the critical rule to enforce, and the failure mode if skipped. Applicable to semantic models, lakehouses, and agent configurations.

Practice Area Critical Rule Failure Mode if Skipped
01
Schema Design
Use business-friendly names everywhere. Tables named by action (e.g. ClaimSubmissions, MemberEnrollment). Columns spelled out fully. Acronyms removed. Units explicit.
Rule
The AI queries metadata, not data. If the name is ambiguous, the DAX generation tool will choose incorrectly — silently.
If Skipped
Agent returns plausible but wrong answers. Users trust them. Trust is harder to rebuild than it was to build.
02
AI Data Schema
Scope the AI schema to the minimum required tables. Exclude lookup-only tables, system columns, foreign keys, and any column not needed to answer business questions.
Rule
Smaller, precise schema = higher accuracy and lower latency. A scoped schema also limits the AI's blast radius if it misinterprets a query.
If Skipped
AI picks the wrong column when similar names exist. Ambiguity in large models causes hallucinated joins and incorrect aggregations.
03
Table & Column Descriptions
Write plain-language descriptions for every table and measure in scope. Define what the table represents, its grain, how it relates to other domains, and any business rules embedded in it.
Rule
Descriptions are the system prompt for the data model. Investing here benefits Copilot, Data Agent, and all future AI surfaces simultaneously.
If Skipped
AI cannot distinguish between similarly named tables (e.g. Claims vs ClaimAdjustments). Response quality degrades proportionally to model complexity.
04
AI Instructions
Write domain-specific AI instructions before connecting the agent. Define: which tables answer which question types, how to handle ambiguous terminology, what to do when data is insufficient.
Rule
Instructions added at the agent level do not override semantic model instructions for DAX generation. Always configure Prep for AI in the semantic model first.
If Skipped
Agent routes financial metric questions to raw lakehouse tables instead of the governed semantic model. Unaudited answers reach business users.
05
Verified Answers
Pin the 10–15 most critical questions to pre-approved visuals. These are the questions compliance officers, executives, or auditors will ask. Verified answers guarantee the response is human-approved.
Rule
Use complete question phrases as triggers — not partial keywords. Verified Answers require Import or DirectQuery mode. Not available in DirectLake.
If Skipped
The agent improvises answers to high-stakes questions. In regulated contexts, an improvised answer to "what is our current HIPAA compliance status" is a liability event.
06
Access & Security Alignment
Align agent data source permissions to the same role structure as the semantic model. Test each role explicitly — verify the agent cannot return data the role cannot see in a report.
Rule
When adding a semantic model to a Data Agent, share access to underlying data sources explicitly — it is not inherited automatically from report permissions.
If Skipped
Users gain data access through the AI surface that they don't have through the reporting surface. This is a HIPAA and PCI DSS violation, not a UX problem.
07
Time Intelligence & Measures
Add pre-built time intelligence measures — year-over-year, month-over-month, rolling averages — before enabling AI. Without them, the agent describes change but cannot explain it.
Rule
Copilot and Data Agents cannot infer trend analysis from raw date columns. The measure must exist in the model for the AI to use it correctly.
If Skipped
Executives ask "why did claims volume drop in Q3?" and the agent returns the Q3 number without context. The answer is technically correct and operationally useless.
Sequencing principle
Steps 01–03 are prerequisites for Steps 04–07. There is no shortcut: AI Instructions and Verified Answers built on poorly named, poorly scoped, undescribed tables produce unpredictable responses at exactly the moments when predictability matters most.
Healthcare & Financial Services · This Client

Four AI use cases — anchored to their three pain points

Each use case maps to a confirmed client pain point. The readiness gate at the bottom of each card is the data work that must be done before the AI layer is credible.

Healthcare · Pain Point 03
Clinical Document Q&A Agent
GA
Problem
Clinicians cannot find answers across EHR notes, discharge summaries, and policy documents. Queries go to IT. Turnaround is days.
How it works
Fabric Data Agent connects to a governed semantic model of curated clinical data. Users ask in plain language. Purview sensitivity labels control what each role can retrieve. Responses cite source documents.
AI data prep
AI Data Schema scoped to patient-safe columns only. AI Instructions define PHI boundaries — agent refuses queries that would surface direct identifiers. Verified Answers for the 10 most common compliance queries.
Outcome
Hours → seconds. PHI stays classified. Full audit trail on every retrieval. HIPAA-defensible by design.
Readiness gate: PHI masked in Silver · Labels enforced · DQ ≥95% on clinical domain
Financial Services · Pain Point 01
Real-Time Fraud & Risk Scoring
GA
Problem
Fraud detection is batch-based with 24-hour lag. By the time patterns are identified, exposure has already occurred and chargebacks are mounting.
How it works
Streaming ingestion feeds a predictive model that scores each transaction in near real-time. Data Activator triggers automated holds below a configurable risk threshold. Risk analysts query the semantic model via Data Agent for investigation context.
AI data prep
Semantic model includes pre-built measures for velocity, behavioral baseline, and anomaly score. AI Instructions route fraud-pattern questions to the analytical model and raw transaction questions to the lakehouse. Time intelligence measures allow trend queries.
Outcome
Detection latency drops dramatically. PCI DSS audit trail automated. Analysts get investigation context in natural language without raw data access.
Readiness gate: Cardholder data scoped · Streaming topology defined · Latency SLA agreed
Cross-Vertical · Pain Point 02
Compliant Self-Service BI with Copilot
GA
Problem
Business users cannot access regulated data directly. Every report request goes to IT with a two-week turnaround. Analysts are a bottleneck, not an asset.
How it works
DirectLake semantic model with RLS enforced at the model layer. Copilot answers natural language questions against the model. Row-level filters ensure each user sees only their authorized population. Sensitivity labels propagate to every Copilot-generated export.
AI data prep
Verified Answers configured for the top 15 executive and compliance questions. AI Instructions distinguish between "member count" (claims-based) and "enrolled members" (eligibility-based) — a common source of conflicting numbers that breaks trust.
Outcome
Analysts become self-sufficient. IT is unblocked. PHI is never exposed. Every AI-generated answer is audit-logged.
Readiness gate: Single semantic model per domain · RLS tested per role · Business glossary agreed
Cross-Vertical · Pain Point 01 + 03
Internal Operations Copilot
Preview → GA FabCon 2026
Problem
Internal teams — compliance, finance, operations — spend hours building one-off data extracts for recurring questions. No single source of truth they can query themselves.
How it works
Multi-source Data Agent connects to the governed semantic model (structured metrics) and a document knowledge base (policies, contracts, SOPs). Teams ask in natural language. Agent routes to the right source based on question type. Responses include citations.
AI data prep
AI Instructions partition the agent: financial metric questions → semantic model, policy questions → document store, operational questions → lakehouse. Example queries provided for each partition so the agent generalizes correctly to similar questions.
Outcome
Compliance teams self-serve audit evidence. Finance teams get consistent metric definitions. Operations teams resolve issues without IT dependency.
Readiness gate: Document corpus curated + labeled · Agent scope defined · Data Agent GA confirmed
The sequencing reality
All four use cases share the same semantic model foundation. Use Cases 2 and 4 can begin scoping immediately — they have the clearest latency SLA and scope boundary. Use Case 1 requires PHI masking in Silver to be complete first. Use Case 3 requires the business glossary conflict (member count definitions) to be resolved before Verified Answers can be configured. Don't attempt all four simultaneously — the foundation work is not parallelizable.
