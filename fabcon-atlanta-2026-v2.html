Core thesis: AI agents are only as reliable as the data you feed them
Every AI announcement traces to grounding agents in trusted, semantically rich enterprise data. Fabric IQ, Ontologies, semantic models, Graph, and MCP are all agent reliability infrastructure.

Fabric IQ + Ontologies

Fabric IQ is the shared semantic framework combining Power BI semantic models + Graph in Fabric. It elevates semantics from BI convenience to AI infrastructure — entities, relationships, rules, actions. Part of Microsoft's IQ layer (alongside Work IQ for M365, Foundry IQ for AI Foundry).

Pre: Semantic models served BI only. No agent context layer.→Fabric IQ: semantic models as AI grounding + graph relationships + operational context

Ontologies Preview — a new workload that enables domain-level entity modeling beyond what semantic models offer. Includes operations agents that monitor the business in real time. Ontologies define the conceptual language that Fabric IQ uses to train agents.

No entity/relationship modeling beyond semantic model measures/dimensions→Ontologies: business entities, properties, rules, actions — agent-readable domain models

Database Hub in Fabric Early Access

A unified database management experience that collapses fragmented Azure Portal blades into a single view. Covers Azure SQL, Cosmos DB, PostgreSQL, MySQL, SQL Server (via Azure Arc), and Fabric Databases — without changing how each service is deployed.

Each database managed in its own Azure Portal blade. No cross-service view.→Single pane: explore, observe, govern, optimize entire estate. Agent-assisted with Copilot insights.

Built-in observability, delegated governance, human-in-the-loop AI agents that surface what changed, explain why, and guide resolution. This is the DBA's new home.

Source: azure.microsoft.com/en-us/blog — "Database Hub in Fabric provides a unified database management experience"

SQL Server 2025 × Fabric Integration

SQL Server 2025 is growing more than 2× as fast as the previous version. The key integration: EXPOSE TO FABRIC, a new T-SQL extension that creates virtualized representations of database objects (tables, views, stored procedures) in OneLake — without moving the underlying data.

ETL pipelines required to move SQL Server data into Fabric analytics layer→EXPOSE TO FABRIC: zero-copy virtualization from on-prem SQL Server into OneLake

Additional: Intelligent Query Processing enhancements work with Fabric's compute engine. Queries federated across SQL Server instances and Spark clusters with optimizer choosing the most efficient execution path. On-premises databases participate via Azure Arc without requiring cloud migration.

Source: windowsnews.ai — "Database administrators can now expose tables, views, and stored procedures directly to Fabric through a new T-SQL extension: EXPOSE TO FABRIC"

SQL Database in Fabric — Next Wave

SQL Database in Fabric reached GA at Ignite November 2025. FabCon Atlanta builds on three pillars driven by customer feedback: Simplified (easier provisioning, management), Autonomous & Secure (self-tuning, built-in security), and Optimized for AI (closer to agents and analytics). Adoption has accelerated since GA.

The strategic question for architects: when to use SQL Database in Fabric vs. Azure SQL vs. SQL Server on-prem/VM? The answer depends on governance requirements, workload proximity to analytics, and whether you need full SQL Server compatibility or can accept the Fabric SaaS model.
